# 项目描述

**Deep Research Agent** — 基于 LangChain DeepAgents 的多智能体深度研究系统，旨在将 LLM 从"聊天助手"升级为"可独立执行长程研究任务的研究智能体"。

## 项目背景

普通 LLM Agent 做长程研究面临三个核心瓶颈：缺乏规划导致维度遗漏、原始搜索结果填满上下文窗口导致信息丢失、单线程无法同时纵深多个方向。本项目利用 LangChain deepagents 框架的四项内置原语（Planning 规划、Context Offloading 上下文卸载、Delegation 委托隔离、Memory 跨 run 记忆）系统性地解决上述问题。

## 系统架构

采用 **Supervisor → Researcher × N → Critic → Verifier** 四层多 Agent 协作架构：

- **Supervisor（研究总监）**：MECE 分解研究课题为互斥穷尽的子问题，为每个 Researcher 编写结构化任务简报（目标/范围/工具优先级/预算），并行委派子智能体执行搜索，归档返回结果到虚拟文件系统，分节撰写结构化报告并最终自评。Supervisor 自身不持有任何搜索工具，全部信息收集通过委托完成。
- **Researcher（研究员）**：遵循 ReAct 行为循环（[REASONING_SUMMARY]→[ACTION]→[OBSERVATION]→[DECISION]），在独立上下文窗口中执行搜索和抓取。每个 Researcher 在代码层有搜索次数硬上限和上下文输出双重限制，返回结构化研究结果（核心发现 + 来源 + 证据强度 + 充分性自评）。
- **Critic（批判审查员）**：对报告初稿做结构性和覆盖度审查，输出评分和缺陷清单（证据薄弱点、未覆盖维度、内部矛盾、引用问题），给出明确的 REQUIRES_REWORK 决策。
- **Verifier（事实核验员）**：读取 /sources/ 下已保存的原始来源文件，逐条核验报告中论断的事实支撑，输出 SUPPORTED / PARTIAL / UNSUPPORTED / CONTRADICTED 判定。

## 技术栈

**后端**：Python 3.12, LangChain DeepAgents, LangGraph (StateGraph 编排), Flask + SSE 实时推流, DuckDuckGo 免费搜索, OpenAlex + Crossref 学术 API, BGE-M3 向量知识库 (Chroma 持久化), BAAI/bge-reranker-v2-m3 跨编码器精排, trafilatura→readability→regex 三级正文抽取, python-docx 报告生成, DeepSeek V4 Pro API (OpenAI 兼容格式)

**前端**：React 19, TypeScript, Vite 6, CSS Modules, highlight.js 代码高亮, tsparticles 粒子背景

## 核心功能

- **多源搜索体系**：DuckDuckGo 通用搜索、OpenAlex/Crossref 英文学术、百度学术/知乎/百度百科国内来源、BGE-M3 本地历史知识库，共 6 类搜索工具
- **搜索质量管线**：多搜回（10 条）→ Cross-Encoder 精排 → 只留 top-4；URL 规范化去重；source_id 全程追踪
- **上下文控制**：全文自动保存到 /sources/ 文件系统，工具仅返回轻量摘要 + source_id；单 Agent 累计输出 18K 软限制 + 30K 硬限制
- **三级运行模式**：Fast（5 次搜索/5 分钟）→ Deep（20 次搜索/不限时）→ Max（+Critic 审查 + Verifier 核验）
- **完整审计追踪**：每次研究独立 run 目录，events.jsonl / sources.jsonl / claims.jsonl 记录全流程，支持断点恢复
- **前端控制台**：实时 SSE 日志流、Agent 活动面板（按子智能体分组）、来源账本（按类型过滤）、论断验证面板（状态高亮）、报告预览与多格式下载（.md / .docx / .txt）
