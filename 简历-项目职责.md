# 项目职责

本项目为个人独立开发项目，负责从零到一的完整设计、实现与迭代。

## 架构设计

- 独立设计 Supervisor + 动态数量 Researcher + Critic + Verifier 四层多 Agent 协作架构，确定各 Agent 的工具边界与通信协议
- 设计 MECE 分解→任务简报→并行委派→归档查漏→分节撰写→Critic 反思→自评归档的完整 7 阶段研究流水线
- 确定 Prompt 工程策略：STATIC/DYNAMIC 拆分以利用 LLM 前缀缓存，渐进式 Skill 加载避免系统 Prompt 膨胀

## 核心功能实现

- 实现 6 类搜索工具（DuckDuckGo 通用、OpenAlex 英文学术、Crossref 元数据、百度学术/知乎/百度百科 国内来源、本地 BGE-M3 向量知识库），含搜索预算代码层硬拦截、按 Agent 隔离的查询去重缓存、上下文输出双重限制
- 搭建"多搜→精排→少留"两段式检索管线，集成 BAAI/bge-reranker-v2-m3 跨编码器对搜索结果按查询相关性重排
- 实现 URL 规范化（移除 utm_*/fbclid 等追踪参数）、source_id 稳定生成（SHA-256 哈希）、全文保存与轻量返回机制
- 将 Researcher 从 OODA 循环升级为 ReAct 行为协议，要求每条发现标注 evidence_strength 并绑定 source_id
- 实现 Verifier 事实核验 Agent，读取原始来源文件逐条核验报告论断

## 后端工程化

- 全配置驱动架构（config.py 160+ 项可调参数），CLI 参数覆盖运行时配置，所有模块使用 cfg.X 动态引用避免 Python 导入时值捕获 bug
- 实现独立 run 目录隔离与断点恢复系统（events.jsonl / sources.jsonl / claims.jsonl），支持通过 --run-id / --resume 管理研究生命周期
- 实现 Flask SSE 桥接服务，将 Agent stdout 实时推流到前端，含 12 个 REST API 端点（启动/停止/跑列表/事件/来源/论断/报告/下载/版本）
- DeepSeek V4 Pro API 并发限流保护（threading.Semaphore）+ 按角色分派推理深度（high/max）
- BGE-M3 本地向量知识库（Chroma 持久化），实现 dense+sparse+RRF 混合检索与 LLM 上下文增强检索
- 国内环境适配：HuggingFace 镜像加速、中科院 arXiv 镜像、百度学术/知乎/百度百科中文来源集成

## 前端控制台

- 独立完成前端从 Vue 3 到 React 19 + TypeScript + Vite 6 + CSS Modules 的整体迁移
- 实现标签页式面板布局
- 集成 Git commit 版本标签、粒子动画背景、三级努力度选择器、终止按钮

## 工程规范

- Git 版本控制，规范化 commit message，按功能原子提交
- 项目配置与文档完整（CLAUDE.md / TECHNICAL.md / BUILD_GUIDE.md）
- 代码层硬约束优先于 Prompt 软引导的设计原则
