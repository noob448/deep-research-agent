# 来源质量准则

## 优先采用（高质量来源）
- 同行评审的学术论文（arXiv、Semantic Scholar、正式期刊）
- 官方文档和技术规范（框架/产品的官方 docs）
- 政府/机构发布的统计数据和报告
- 知名技术媒体的一手报道（如 IBM Developer、NVIDIA Blog、Anthropic Blog）
- 有明确作者、发表日期和出处的深度技术文章

## 保持警惕（降权或舍弃）
- 无发表日期的内容
- 内容农场、SEO 优化明显的页面
- 营销软文、产品推销（"某某产品是最好选择"式文章）
- 缺乏出处的二手转述、洗稿文章
- 论坛帖子和个人博客（除非作者是公认领域专家）

## 内部历史研究的使用

- 知识库返回的「内部历史研究·已浓缩」是过去某时点的结论，可能已过时。
- 将其作为背景和起点，不作为最终事实；时效性强的结论必须用新搜索交叉验证。
- 引用时与外部来源区分标注（例如"据本项目历史研究"），不与外部一手来源混为一谈。

## 处理规则

1. **交叉验证**：关键论断必须跨 ≥2 个独立来源验证后才能写入笔记。
2. **标注存疑来源**：来源可信度存疑时，在笔记里显式标注「该来源可信度较低」，不要直接当作事实。
3. **记录分歧**：多个来源冲突时，记录分歧本身并指出各自来源，不要单方面下结论。
4. **宁缺毋滥**：舍弃与子问题无关但"看起来相关"的内容。宁可笔记短而精，不要长而杂。
5. **时效优先**：优先引用近 2 年内的资料。超过 3 年的技术文章需标注年份并提示可能过时。
