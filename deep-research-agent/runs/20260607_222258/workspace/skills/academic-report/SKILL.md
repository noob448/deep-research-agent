# 学术研究报告格式

## 报告结构

最终报告写入 `/report.md`，按以下结构组织：

```markdown
# [研究主题标题]

## 摘要
- 3-5 句话概述研究问题和主要发现
- 读者看完摘要就能判断这份报告是否与自己相关

## 研究背景
- 为什么这个问题重要
- 已有的共识和争论
- 本报告的研究范围

## 核心发现
- 按主题分组，每个主题一个三级标题
- 每个发现用要点或段落呈现
- 不同来源的信息交叉验证

## 分析与讨论
- 不同来源的矛盾之处
- 信息的局限性（时效性、来源偏见等）
- 尚待进一步研究的问题

## 结论
- 对研究问题的直接回答
- 关键要点回顾
- 行动建议（如果适用）

## 参考来源
- 按主题分组列出所有引用来源
- 每个来源包含：标题、URL、简要说明
```

## 引用规则（编号制）

**核心原则：每篇论文/每个独立来源单独编号，禁止将多篇论文合并为一个引用。**

- **正文中**：每条事实声明后标注编号 `[1]`、`[2]`、`[3]`……同一来源多次引用时复用同一编号。
- **禁止打包引用**：`[1]` 不能同时包含"BLIP-2 (arXiv:xxx) + LLaVA (arXiv:yyy) + Flamingo + Chameleon"——必须拆为独立的 `[1]`、`[2]`、`[3]`、`[4]`。
- **学术论文**：必须注明 arXiv ID 或 DOI（如 `arXiv:2301.12597`、`doi:10.1109/...`），方便读者直接定位原文。
- **博客/网页**：注明作者/机构 + URL。

**报告末尾**：添加「参考来源」章节，按编号列出所有来源。
```
## 参考来源
[1] 论文标题 - 作者 (arXiv:xxxx 或 doi:xxxx)
[2] 文章标题 - 作者/机构 (URL)
```

示例：
> 记忆系统可分为短期记忆和长期记忆两大类 [1]。其中长期记忆又可细分为情景记忆、语义记忆和程序记忆 [2][3]。
>
> ## 参考来源
> [1] Memory in AI Agents: A Comprehensive Guide - IBM Developer (https://...)
> [2] Episodic Memory for AI Agents - Atlan (https://...)
> [3] The Four Types of Memory - Art of Smart (https://...)

## 质量检查清单

写完报告后逐项检查：

- [ ] 摘要是否涵盖了主要发现？
- [ ] 每条事实声明是否有编号引用？
- [ ] **是否每条引用只对应一个独立来源（无打包引用）？**
- [ ] 学术论文引用是否标注了 arXiv ID 或 DOI？
- [ ] 报告末尾是否有完整的「参考来源」列表（编号对应正文）？
- [ ] 不同来源的信息是否交叉验证过？
- [ ] 是否明确标注了信息的局限性？
- [ ] 结论是否直接回答了研究问题？
- [ ] 报告结构是否完整（不缺少任何必需章节）？
